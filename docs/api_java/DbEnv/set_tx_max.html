Berkeley DB: DbEnv.set_tx_max

DbEnv.set_tx_max

import com.sleepycat.db.*;


public void set_tx_max(int tx_max);

Description

Set the maximum number of simultaneous transactions that are supported. This bounds the size of backing files and is used to derive limits for the size of the lock region and logfiles. When there are more than the specified number of concurrent transactions, calls to DbTxnMgr.begin may cause backing files to grow. If no value is specified, a default value is used.
