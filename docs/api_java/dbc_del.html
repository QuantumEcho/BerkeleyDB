Dbc.del

 

import com.sleepycat.db.*;


public int del(int flags)
	throws DbException;

Description

The Dbc.del method deletes the key/data pair currently referenced by the cursor.

The flags parameter is currently unused, and must be set to 0.

The cursor position is unchanged after a delete, and subsequent calls to cursor functions expecting the cursor to reference an existing key will fail.

If the element has already been deleted, Dbc.del will return DB_KEYEMPTY.

If the cursor is not yet initialized, the Dbc.del method throws an exception that encapsulates EINVAL.

Otherwise, the Dbc.del method throws an exception that encapsulates a non-zero error value on failure.

Errors

If a fatal error occurs in Berkeley DB, the Dbc.del method will fail and throw a DbRunRecoveryException, at which point all subsequent database calls will fail in the same way.

In addition, if the operation was selected to resolve a deadlock, the Dbc.del method will fail and throw a DbDeadlockException exception.

In addition, the Dbc.del method may fail and throw an exception encapsulating a non-zero error for the following conditions:

EINVAL
An invalid flag value or parameter was specified.

In addition, the Dbc.del method may fail and throw an exception for errors specified for other Berkeley DB and C library or system methods.

Dbc
